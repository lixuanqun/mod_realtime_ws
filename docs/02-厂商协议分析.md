# 02 · 厂商协议分析与适配差异

本文对比三家厂商的 WebSocket 协议，识别共性与差异，作为 Provider Adapter 抽象的依据。

## 1. 总览对比

| 维度 | GLM-Realtime | 阿里云 ISI (SpeechTranscriber) | 豆包 / 火山 端到端实时语音 |
| --- | --- | --- | --- |
| 形态 | 端到端 S2S | **流式 ASR（非端到端）** | 端到端 S2S |
| 接入地址 | `wss://open.bigmodel.cn/api/paas/v4/realtime` | `wss://nls-gateway-<region>.aliyuncs.com/ws/v1` | `wss://openspeech.bytedance.com/api/v3/realtime/dialogue`（以官方文档为准） |
| 传输编码 | **文本帧 JSON**（音频 base64 内嵌） | **文本帧 JSON**（控制）+ **二进制帧**（音频） | **全二进制帧**（自定义封帧 + JSON/gzip payload） |
| 事件标识 | `type` 字符串（OpenAI Realtime 风格） | `header.namespace` + `header.name` | 二进制 header 内 `event` 数字 ID |
| 鉴权 | Header `Authorization: Bearer <JWT/APIKey>` | URL/Header `X-NLS-Token` + `appkey` | 火山签名（AK/SK）或 Token，握手 Header |
| 输入音频 | wav/pcm，单声道 16bit，pcm16(16k)/pcm24 | pcm/opus/wav，8k/16k 单声道 16bit | pcm，采样率由 StartSession 指定 |
| 输出音频 | **pcm 24kHz 单声道 16bit** | 无（ASR 无音频输出，需另接 TTS） | pcm（采样率由 TTS 配置指定） |
| VAD/打断 | server_vad / client_vad，服务端可托管打断 | 服务端断句（SentenceBegin/End），无打断概念 | 仅 server_vad，服务端托管 |
| 转写文本 | `response.audio_transcript.delta` + 输入转写 | `TranscriptionResultChanged`/`SentenceEnd`（核心输出） | `ASRResponse` |
| Function Call | 支持（`tools` + `function_call_arguments`） | 不支持 | 视版本支持（以文档为准） |
| 压缩 | 无（JSON） | 无 | 支持 gzip（payload） |
| 心跳 | 服务端 `heartbeat` 事件 | 依赖 ws ping/pong | 依赖 ws ping/pong |

> **关键结论**：三家协议差异巨大（字符串事件 / namespace+name / 二进制数字事件 ID），且阿里云 ISI 本身只是 ASR，要做到"端到端语音对话"必须由 router 编排 `ASR(阿里云) → LLM(自选) → TTS(阿里云 CosyVoice/或他家)`。因此 **Adapter 抽象必须同时支持"真端到端"与"组合式端到端"两类后端**。

## 2. GLM-Realtime 协议要点

OpenAI Realtime 风格，所有交互为 JSON 文本帧，事件用 `type` 区分，音频为 base64 字符串内嵌。

### 2.1 客户端事件（→ 服务端）

| 事件 type | 用途 |
| --- | --- |
| `session.update` | 配置会话：model、voice、modalities、input/output_audio_format、turn_detection、tools、beta_fields(chat_mode/greeting) |
| `input_audio_buffer.append` | 上传音频（base64，推荐 100ms/帧，≤50QPS） |
| `input_audio_buffer.commit` | 提交音频（client_vad 模式需要） |
| `input_audio_buffer.clear` | 清空输入缓冲 |
| `response.create` | 触发推理（client_vad 模式） |
| `response.cancel` | **打断**：取消进行中的响应 |
| `conversation.item.create` | 文本输入 / 回填 function_call_output |

### 2.2 服务端事件（→ 客户端）

| 事件 type | 用途 |
| --- | --- |
| `session.created` / `session.updated` | 会话就绪/更新确认 |
| `input_audio_buffer.speech_started` / `speech_stopped` | server_vad 检测说话开始/结束（**打断触发源**） |
| `input_audio_buffer.committed` | 音频提交确认 |
| `conversation.item.input_audio_transcription.completed` | 用户音频转写完成 |
| `response.created` / `response.done` | 一轮响应开始/结束 |
| `response.audio.delta` / `response.audio.done` | **下行音频**（base64 pcm24k）流式/结束 |
| `response.audio_transcript.delta` / `.done` | AI 语音对应文本（字幕） |
| `response.text.delta` / `.done` | 文本响应 |
| `response.function_call_arguments.done` | 工具调用参数就绪 |
| `rate_limits.updated` | 速率限制更新 |
| `heartbeat` | 保活 |
| `error` | 错误（多数可恢复） |

### 2.3 打断流程

- server_vad 模式：`turn_detection.interrupt_response=true` 时服务端自动打断；客户端收到 `speech_started` 即可清空本地下行缓冲。
- 也可主动发 `response.cancel`。

## 3. 阿里云 ISI（SpeechTranscriber）协议要点

控制为 JSON 文本帧（`header`+`payload`），音频为二进制帧。**仅 ASR**。

### 3.1 指令（客户端 →）

| name (namespace=SpeechTranscriber) | 用途 |
| --- | --- |
| `StartTranscription` | 启动识别，payload: format/sample_rate/enable_intermediate_result/enable_punctuation_prediction/enable_inverse_text_normalization 等 |
| `StopTranscription` | 结束识别 |
| （二进制帧） | 上传音频流 |

Header 统一字段：`message_id`、`task_id`、`namespace`、`name`、`appkey`（请求）；响应含 `status`、`status_message/status_text`、`task_id`。

### 3.2 事件（服务端 →）

| name | 用途 |
| --- | --- |
| `TranscriptionStarted` | 识别已开始 |
| `SentenceBegin` | 检测到一句话开始（payload: index/time）→ **可作为打断触发** |
| `TranscriptionResultChanged` | 中间识别结果（payload: index/time/result/words） |
| `SentenceEnd` | 一句话结束（含最终 result）→ 触发下游 LLM+TTS |
| `TranscriptionCompleted` | 整个识别任务结束 |
| `TaskFailed` | 失败 |

### 3.3 端到端编排（router 内组合）

由于 ISI 只有 ASR，router 需将其包装成"伪 S2S"：

```
用户音频 ─► ISI(ASR) ─SentenceEnd─► LLM(自选/可配置) ─► TTS(阿里云CosyVoice 或他家) ─► 下行音频
            │
            └ SentenceBegin / 本地VAD ─► 打断
```

- 该 Adapter 需要额外的 LLM 客户端与 TTS 客户端（均可配置）。
- 时延高于真端到端，定位为"兼容/降级"通道。

## 4. 豆包 / 火山 端到端实时语音协议要点

全二进制自定义封帧协议，**大端序**。

### 4.1 二进制封帧

```
+-------------------+------------------+-------------------+----------+
| 4-byte Header     | optional fields  | Payload Size (4B) | Payload  |
+-------------------+------------------+-------------------+----------+

Header(4字节):
  byte0: 高4位 protocol version | 低4位 header size(以4字节为单位)
  byte1: 高4位 message type     | 低4位 message type specific flags
  byte2: 高4位 serialization    | 低4位 compression
  byte3: reserved
```

- message type：`0b0001` full client request、`0b0010` audio-only client、`0b1001` full server response、`0b1011` audio-only server、`0b1111` error。
- serialization：JSON；compression：gzip（payload）。
- message type specific flags：携带 event 编号等（WithEvent）。
- 含可选字段：`event`(int32)、`session_id`(带长度前缀字符串)、`connect_id` 等。

### 4.2 事件 ID（部分）

| 方向 | event | 名称 | 用途 |
| --- | --- | --- | --- |
| 上行 | 1 | StartConnection | 建立连接 |
| 上行 | 2 | FinishConnection | 关闭连接 |
| 上行 | 100 | StartSession | 开始会话（配置 TTS speaker/audio_params、ASR、对话参数） |
| 上行 | 102 | FinishSession | 结束会话（连接可复用） |
| 上行 | 200/300 区段 | TaskRequest / 上传音频 | 上传用户音频 |
| 上行 | — | ChatTTSText / SayHello 等 | 文本注入/欢迎语（视版本） |
| 下行 | 50 | ConnectionStarted | 连接已建立 |
| 下行 | 150 | SessionStarted | 会话已建立 |
| 下行 | 350 | TTSSentenceStart | 一句 TTS 开始 |
| 下行 | 351 | TTSSentenceEnd | 一句 TTS 结束 |
| 下行 | 352 | **TTSResponse** | **下行音频数据（payload 为二进制音频）** |
| 下行 | 359 | TTSEnded | TTS 结束 |
| 下行 | 450 | ASRInfo | ASR 开始（可作打断触发） |
| 下行 | 451 | ASRResponse | ASR 识别结果 |
| 下行 | 459 | ASREnded | ASR 结束 |
| 下行 | 5xx | ChatResponse / ConversationTruncated 等 | 对话/截断 ack |
| 下行 | 153/152 | SessionFinished/SessionFailed | 会话结束/失败 |

> 注：具体 event 数值以火山官方最新文档为准，Adapter 内集中维护一份常量映射表，便于跟随官方调整。

### 4.3 打断流程

- 仅 server_vad。服务端检测到用户说话会下发 ASRInfo，客户端据此清空下行缓冲；可发送截断/打断请求停止当前 TTS。

## 5. 统一抽象：Provider Adapter 接口

为屏蔽差异，router 定义统一 Adapter 接口（伪代码，Go 风格）：

```go
type Provider interface {
    // 建立到厂商的连接并完成握手/鉴权
    Connect(ctx context.Context, cfg SessionConfig) error
    // 配置会话（音色/指令/VAD/工具）。组合式后端在此初始化 ASR+LLM+TTS
    UpdateSession(cfg SessionConfig) error
    // 上行：推送用户音频帧（PCM 16k 单声道 16bit，统一内部格式）
    PushAudio(frame []byte) error
    // 上行控制：提交/清空/触发响应/打断/文本输入/回填工具结果
    Commit() error
    Clear() error
    CreateResponse() error
    Cancel() error                 // 打断
    SendText(text string) error
    SubmitToolResult(callID string, output string) error
    // 下行：统一事件回调（见 03 文档的内部事件定义）
    Events() <-chan ProviderEvent
    Close() error
}

// 统一下行事件（厂商事件归一化后）
type ProviderEvent struct {
    Type        EventType   // SessionReady/SpeechStarted/SpeechStopped/
                            // UserTranscript/AudioDelta/AudioDone/
                            // AssistantTranscript/ResponseDone/ToolCall/Error/Heartbeat
    Audio       []byte      // AudioDelta 时为 PCM（已归一化采样率）
    Text        string
    SampleRate  int
    ToolCall    *ToolCall
    Err         *ProviderError
    Raw         json.RawMessage // 透传原始，便于排障
}
```

各 Adapter 内部完成：

| 任务 | GLM | 阿里云 | 豆包 |
| --- | --- | --- | --- |
| 封帧 | JSON 文本 | JSON 文本 + 二进制音频 | 二进制 header + gzip JSON |
| 音频上行编码 | base64→JSON | 二进制帧 | 二进制 audio-only 帧 |
| 音频下行解码 | base64(pcm24k) | （无，来自 TTS） | TTSResponse 二进制 |
| 事件归一化 | `type` 映射 | `namespace.name` 映射 | `event` 数字映射 |
| 鉴权 | JWT/APIKey | Token+appkey | 火山签名 |
| 打断 | response.cancel / speech_started | SentenceBegin/本地VAD | ASRInfo/截断请求 |
| 采样率归一 | 24k→内部 | — | TTS率→内部 |

## 6. 统一事件归一化映射表

| 统一事件 (内部) | GLM | 阿里云 ISI | 豆包 |
| --- | --- | --- | --- |
| SessionReady | session.created/updated | TranscriptionStarted | SessionStarted(150) |
| SpeechStarted（打断源） | input_audio_buffer.speech_started | SentenceBegin | ASRInfo(450) |
| SpeechStopped | input_audio_buffer.speech_stopped | SentenceEnd | ASREnded(459) |
| UserTranscript | conversation.item.input_audio_transcription.completed | SentenceEnd.result / TranscriptionResultChanged | ASRResponse(451) |
| AudioDelta（下行音频） | response.audio.delta | (TTS音频) | TTSResponse(352) |
| AudioDone | response.audio.done | (TTS结束) | TTSEnded(359) |
| AssistantTranscript | response.audio_transcript.delta/done | (LLM文本) | ChatResponse |
| ResponseDone | response.done | (一轮结束) | TTSEnded/SessionFinished |
| ToolCall | response.function_call_arguments.done | N/A | ChatResponse(tool) |
| Heartbeat | heartbeat | (ws pong) | (ws pong) |
| Error | error | TaskFailed | error(0b1111) |

这张表是 Adapter 实现的"契约"，新增厂商时只需补一列。
