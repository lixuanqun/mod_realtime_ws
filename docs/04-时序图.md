# 04 · 时序图

下列时序图覆盖核心交互流程。参与方：
- **Caller**：主叫用户（电话/SIP/WebRTC）
- **FS+mod**：FreeSWITCH 通道 + mod_realtime_ws
- **Router**：realtime-router
- **Vendor**：语音大模型厂商

## 1. 建链与会话初始化（以 GLM server_vad 为例）

```mermaid
sequenceDiagram
    participant Caller
    participant FSmod as FS+mod_realtime_ws
    participant Router as realtime-router
    participant Vendor as GLM-Realtime

    Caller->>FSmod: 呼入/接通 (媒体建立)
    Note over FSmod: dialplan 执行 application=realtime_ws
    FSmod->>Router: WS 连接 (X-RT-Provider=glm, profile)
    FSmod->>Router: session.start {provider,profile,vars}
    Router->>Vendor: WS 连接 (Authorization: Bearer JWT)
    Vendor-->>Router: session.created
    Router->>Vendor: session.update {voice,vad,tools,greeting,output=pcm}
    Vendor-->>Router: session.updated
    Router-->>FSmod: session.ready {output_sample_rate=24000}
    Note over FSmod: media bug 启动，开始采集上行音频
    opt 开场白
        Vendor-->>Router: response.audio.delta (greeting)
        Router-->>FSmod: [二进制音频帧] 下行
        FSmod-->>Caller: 播放问候语
    end
```

## 2. 正常一轮对话（server_vad，端到端）

```mermaid
sequenceDiagram
    participant Caller
    participant FSmod as FS+mod
    participant Router
    participant Vendor

    loop 每 20ms
        Caller->>FSmod: 说话 (PCM)
        FSmod->>Router: [二进制音频帧] uplink (聚合为100ms)
        Router->>Vendor: input_audio_buffer.append (base64)
    end
    Vendor-->>Router: input_audio_buffer.speech_started
    Router-->>FSmod: speech.started
    Caller->>FSmod: 停止说话
    Vendor-->>Router: speech_stopped / committed
    Router-->>FSmod: speech.stopped
    Vendor-->>Router: conversation.item.input_audio_transcription.completed
    Router-->>FSmod: user.transcript {text, is_final:true}
    Vendor-->>Router: response.created
    Router-->>FSmod: response.start {response_id}
    par 流式下行
        Vendor-->>Router: response.audio.delta (pcm24k base64)
        Router->>Router: 解码+重采样(24k→8k/16k)
        Router-->>FSmod: [二进制音频帧] downlink
        FSmod-->>Caller: 播放 AI 语音
    and 字幕
        Vendor-->>Router: response.audio_transcript.delta
        Router-->>FSmod: assistant.transcript {is_final:false}
    end
    Vendor-->>Router: response.audio.done / response.done
    Router-->>FSmod: response.end {response_id}
```

## 3. 打断 barge-in（双路径）

```mermaid
sequenceDiagram
    participant Caller
    participant FSmod as FS+mod
    participant Router
    participant Vendor

    Note over FSmod,Caller: AI 正在播报中...
    Caller->>FSmod: 用户突然开口
    rect rgb(235,245,255)
    Note right of FSmod: 快路径(可选)
    FSmod->>FSmod: 本地VAD命中 → 暂停/降低下行播放
    FSmod->>Router: barge_in.detected
    end
    FSmod->>Router: [二进制音频帧] uplink (用户新语音)
    Router->>Vendor: input_audio_buffer.append
    Vendor-->>Router: input_audio_buffer.speech_started
    rect rgb(255,240,235)
    Note right of Router: 准路径(权威)
    Router->>Vendor: response.cancel (打断当前响应)
    Router-->>FSmod: clear_playback {reason:barge_in}
    FSmod->>FSmod: 清空下行 jitter buffer (真正静音)
    end
    Vendor-->>Router: response.cancelled
    Note over FSmod,Vendor: 进入下一轮，处理用户新语音
```

> 消抖：若快路径触发后在 `barge_in_confirm_timeout`(默认 400ms) 内未收到厂商 `speech_started`，router 不发 `clear_playback`，mod 恢复播放，避免噪声误打断。

## 4. Function Call（工具调用）

```mermaid
sequenceDiagram
    participant Caller
    participant FSmod as FS+mod
    participant Router
    participant Vendor
    participant Biz as 业务Webhook

    Caller->>FSmod: "帮我查一下订单12345"
    FSmod->>Router: [音频帧]
    Router->>Vendor: append/commit
    Vendor-->>Router: response.function_call_arguments.done {name:query_order,args}
    Router-->>FSmod: tool.call {call_id,name,arguments}
    alt router 直接编排(推荐)
        Router->>Biz: POST /tools/query_order {args}
        Biz-->>Router: {result}
    else 交给 mod/上层
        FSmod->>FSmod: 执行/ESL 回调业务
        FSmod->>Router: tool.result {call_id,output}
    end
    Router->>Vendor: conversation.item.create (function_call_output)
    Router->>Vendor: response.create (继续生成)
    Vendor-->>Router: response.audio.delta ...
    Router-->>FSmod: [音频帧] 下行
    FSmod-->>Caller: 播报查询结果
```

## 5. 阿里云 ISI 组合式端到端（ASR+LLM+TTS）

```mermaid
sequenceDiagram
    participant Caller
    participant FSmod as FS+mod
    participant Router
    participant ASR as 阿里云ISI(ASR)
    participant LLM as LLM(可配置)
    participant TTS as TTS(CosyVoice/他家)

    FSmod->>Router: session.start {provider:aliyun}
    Router->>ASR: WS连接 + StartTranscription
    ASR-->>Router: TranscriptionStarted
    Router-->>FSmod: session.ready
    loop 用户说话
        FSmod->>Router: [音频帧]
        Router->>ASR: [二进制音频]
        ASR-->>Router: TranscriptionResultChanged (中间结果)
        Router-->>FSmod: user.transcript {is_final:false}
    end
    ASR-->>Router: SentenceEnd {result:最终文本}
    Router-->>FSmod: user.transcript {is_final:true}
    Router->>LLM: 请求(流式)
    LLM-->>Router: token 流
    Router->>TTS: 流式合成
    TTS-->>Router: 音频流
    Router-->>FSmod: [音频帧] 下行
    FSmod-->>Caller: 播报
    Note over Router: 打断: SentenceBegin/本地VAD → 停止LLM/TTS + clear_playback
```

## 6. 豆包(火山) 端到端（二进制协议）

```mermaid
sequenceDiagram
    participant FSmod as FS+mod
    participant Router
    participant Vendor as 豆包/火山

    FSmod->>Router: session.start {provider:doubao}
    Router->>Vendor: [二进制] StartConnection(event=1)
    Vendor-->>Router: ConnectionStarted(50)
    Router->>Vendor: [二进制] StartSession(100) {tts:speaker/audio_params, asr, dialog}
    Vendor-->>Router: SessionStarted(150)
    Router-->>FSmod: session.ready
    loop 上行
        FSmod->>Router: [音频帧]
        Router->>Vendor: [二进制] audio-only client request
    end
    Vendor-->>Router: ASRInfo(450) / ASRResponse(451)
    Router-->>FSmod: speech.started / user.transcript
    Vendor-->>Router: TTSSentenceStart(350)
    Vendor-->>Router: TTSResponse(352) [二进制音频]
    Router-->>FSmod: [音频帧] 下行
    Vendor-->>Router: TTSSentenceEnd(351) / TTSEnded(359)
    Router-->>FSmod: response.end
    Note over Router: 打断: 收到新的 ASRInfo → 截断请求 + clear_playback
```

## 7. 异常与重连

```mermaid
sequenceDiagram
    participant FSmod as FS+mod
    participant Router
    participant Vendor

    Note over Router,Vendor: 厂商连接异常断开
    Vendor--xRouter: ws closed / error
    Router->>Router: 标记会话 degraded, 启动指数退避重连
    Router-->>FSmod: session.error {recoverable:true}
    opt 播放等待提示
        Router-->>FSmod: [音频帧] "请稍候" 提示音(本地)
    end
    loop 退避重连 1s,2s,4s...(上限)
        Router->>Vendor: 重新 Connect + session.update(恢复配置)
    end
    alt 重连成功
        Vendor-->>Router: session ready
        Router-->>FSmod: session.ready (resume)
    else 超过最大重试 / 不可恢复
        Router-->>FSmod: session.error {recoverable:false}
        Note over FSmod: 触发降级(切换厂商) 或 转人工/挂机
    end
```

## 8. 挂机与资源回收

```mermaid
sequenceDiagram
    participant Caller
    participant FSmod as FS+mod
    participant Router
    participant Vendor

    Caller->>FSmod: 挂机 (CHANNEL_HANGUP)
    FSmod->>FSmod: 移除 media bug, 停止采集
    FSmod->>Router: session.stop {reason:hangup}
    FSmod->>Router: WS Close
    Router->>Vendor: 结束会话 (FinishSession/StopTranscription/ws close)
    Router->>Router: 落地指标/录音, 释放会话资源
    Vendor-->>Router: 确认/关闭
```
